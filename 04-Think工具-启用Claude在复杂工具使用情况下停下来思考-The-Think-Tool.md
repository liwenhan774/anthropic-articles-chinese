# "Think"工具：启用Claude在复杂工具使用情况下停下来思考

> **原文链接**: https://www.anthropic.com/engineering/claude-think-tool  
> **发布日期**: 2025年3月20日

---

## 更新说明 (2025年12月15日)

扩展思考功能自最初发布以来已有改进，因此我们建议在大多数情况下使用该功能而不是专用的think工具。扩展思考提供类似的益处——给Claude空间来推理复杂问题——但具有更好的集成和性能。

---

## 摘要

随着我们继续增强Claude的复杂问题解决能力，我们发现了一种特别有效的方法：**"think"工具**，它在复杂任务期间为结构化思考创造专用空间。

这种简单而强大的技术与Claude的新"扩展思考"功能不同——在复杂工具使用情况下显著提高了Claude的智能体工具使用能力，包括遵循策略、做出一致决策和处理多步骤问题，且实现开销最小。

---

## 什么是"Think"工具？

使用"think"工具，我们让Claude能够在得出最终答案的过程中包含一个额外的思考步骤——拥有自己指定的空间。

虽然听起来类似于扩展思考，但这是一个不同的概念：
- **扩展思考**：Claude在开始生成响应之前进行的内容，深度考虑并迭代其计划
- **Think工具**：Claude在生成响应后添加的步骤，用于停下来思考是否有足够的信息继续前进

**Think工具更适合**：
- Claude无法仅从用户查询中获得所有信息来制定响应的情况
- 需要处理外部信息（如工具调用结果中的信息）
- 长工具调用链中需要仔细分析工具输出

**扩展思考更适合**：
- 简单的工具使用场景（如非顺序工具调用或简单指令遵循）
- 不需要Claude调用工具的用例（如编码、数学和物理）

---

## 示例实现

```json
{
  "name": "think",
  "description": "使用这个工具来思考某事。它不会获取新信息或更改数据库，只是将想法附加到日志中。在需要复杂推理或某些缓存记忆时使用它。",
  "input_schema": {
    "type": "object",
    "properties": {
      "thought": {
        "type": "string",
        "description": "要思考的想法。"
      }
    },
    "required": ["thought"]
  }
}
```

## τ-Bench上的性能

我们使用τ-bench（tau-bench）评估了"think"工具，这是一个全面的基准测试，旨在测试模型在真实客户服务场景中使用工具的能力。

### 性能分析

我们的评估比较了几种不同的配置：

1. 基线（无"think"工具，无扩展思考模式）
2. 仅扩展思考模式
3. 仅"think"工具
4. "think"工具+优化提示（针对航空领域）

### 结果

**航空领域**：
- "think"工具+优化提示：pass^1指标达到0.570
- 基线：仅0.370
- **相对改进：54%**

**零售领域**：
- "think"工具：0.812
- 基线：0.783

---

## 关键洞察

1. **提示在困难领域很重要**：简单地提供"think"工具可能会在一定程度上提高性能，但将其与优化提示配对在困难领域产生了显著更好的结果。

2. **跨试验的一致性改进**：使用"think"的改进在pass^k（k最高到5）中都保持，表明该工具帮助Claude更有效地处理边缘案例和不寻常场景。

## SWE-Bench上的性能

类似的"think"工具被添加到我们的SWE-bench设置中，评估Claude 3.7 Sonnet时，帮助达到了最先进的0.623分数。

实验显示（n=30个样本使用"think"工具，n=144个样本不使用），包含该工具平均提高了1.6%的性能。

---

## 何时使用"Think"工具

基于这些评估结果，我们确定了Claude从"think"工具中受益最多的特定场景：

1. **工具输出分析**：当Claude需要仔细处理先前工具调用的输出才能行动，可能需要回溯其方法时
2. **策略密集型环境**：当Claude需要遵循详细指南并验证合规性时
3. **顺序决策**：当每个动作都建立在先前动作之上，且错误代价高昂时（常见于多步骤领域）

---

## 实施最佳实践

### 1. 使用领域特定示例进行策略性提示

最有效的方法是提供关于何时以及如何使用"think"工具的清晰指令。提供针对你特定用例的示例显著提高了模型使用"think"工具的效果：
- 推理过程中期望的细节级别
- 如何将复杂指令分解为可操作的步骤
- 处理常见场景的决策树
- 如何检查是否已收集所有必要信息

### 2. 将复杂指导放在系统提示中

我们发现，当指导较长和/或复杂时，将关于"think"工具的指令放在系统提示中比放在工具描述本身更有效。

---

## 何时不使用"Think"工具

虽然"think"工具可以提供实质性改进，但它并不适用于所有工具使用用例，并且确实以增加提示长度和输出token为代价：

1. **非顺序工具调用**：如果Claude只需要进行一次工具调用或多次并行调用来完成任务，添加"think"不太可能有任何改进
2. **简单指令遵循**：当没有太多约束需要遵守，且其默认行为足够好时，不太可能有额外的"think"带来的收益

---

## 开始使用

"think"工具是对Claude实现的简单添加，只需几个步骤即可产生有意义的改进：

1. **用智能体工具使用场景进行测试**：从具有挑战性的用例开始——Claude目前在策略遵循或复杂推理的长工具调用链中遇到困难的地方
2. **添加工具定义**：实现针对你领域的"think"工具。它需要最少的代码，但支持更结构化的推理
3. **监控和优化**：观察Claude在实践中如何使用该工具，并调整提示以鼓励更有效的思考模式

---

## 结论

我们的研究表明，"think"工具可以显著增强Claude 3.7 Sonnet在需要策略遵循和长工具调用链推理的复杂任务上的性能。"Think"不是万能的解决方案，但它为正确的用例提供了实质性益处，且实现复杂性最小。
