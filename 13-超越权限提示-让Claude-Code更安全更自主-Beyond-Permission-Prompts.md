# 超越权限提示：让Claude Code更安全更自主 (Beyond permission prompts: making Claude Code more secure and autonomous)

> **原文链接**: https://www.anthropic.com/engineering/claude-code-sandboxing  
> **发布日期**: 2025年10月20日  
> **作者**: David Dworken 和 Oliver Weller-Davies

## 摘要

Claude Code的新沙盒功能、bash工具和Claude Code on the web通过启用两个边界减少权限提示并增加用户安全性：文件系统和网络隔离。

在Claude Code中，Claude与你一起编写、测试和调试代码，导航你的代码库，编辑多个文件，并运行命令来验证其工作。给Claude这种对你的代码库和文件的访问权限可能会引入风险，特别是在提示注入的情况下。

为了帮助解决这个问题，我们在Claude Code中引入了两个基于沙盒构建的新功能，这两个功能都旨在为开发者提供一个更安全的工作场所，同时也允许Claude更自主地运行，减少权限提示。在我们的内部使用中，我们发现沙盒安全地将权限提示减少了84%。通过定义Claude可以在其中自由工作的设定边界，它们增加了安全性和自主性。

---

## 在Claude Code上保持用户安全

Claude Code在基于权限的模型上运行：默认情况下，它是只读的，这意味着它在进行修改或运行任何命令之前会请求权限。有一些例外：我们自动允许安全的命令如echo或cat，但大多数操作仍然需要明确批准。

不断点击"批准"会减慢开发周期，并可能导致"批准疲劳"，用户可能不会密切关注他们正在批准的内容，进而使开发变得不那么安全。

为了解决这个问题，我们为Claude Code推出了沙盒功能。

---

## 沙盒：一种更安全、更自主的方法

沙盒创建预定义的边界，Claude可以在其中更自由地工作，而不是为每个动作请求权限。启用沙盒后，你会获得 drastically 更少的权限提示和增加的安全性。

我们的沙盒方法建立在操作系统级功能之上，以启用两个边界：

1. **文件系统隔离**：确保Claude只能访问或修改特定目录。这对于防止提示注入的Claude修改敏感系统文件特别重要。
2. **网络隔离**：确保Claude只能连接到批准的服务器。这防止提示注入的Claude泄露敏感信息或下载恶意软件。

值得注意的是，有效的沙盒需要**同时**具备文件系统和网络隔离。没有网络隔离，受损的智能体可能泄露SSH密钥等敏感文件；没有文件系统隔离，受损的智能体可能轻易逃离沙盒并获得网络访问。通过同时使用这两种技术，我们可以为Claude Code用户提供更安全、更快的智能体体验。

## Claude Code中的两个新沙盒功能

### 沙盒化bash工具：无需权限提示的安全bash执行

我们正在引入一个新的沙盒运行时，作为研究预览版提供测试版，它让你可以精确定义智能体可以访问哪些目录和网络主机，而无需启动和管理容器的开销。这可用于沙盒任意进程、智能体和MCP服务器。

在Claude Code中，我们使用这个运行时来沙盒bash工具，它允许Claude在你设置的定义限制内运行命令。在安全沙盒内，Claude可以更自主地运行并安全地执行命令，无需权限提示。如果Claude尝试访问沙盒**之外**的某些东西，你会立即收到通知，并可以选择是否允许它。

我们构建在OS级原语之上，如Linux bubblewrap和MacOS seatbelt，以在OS级别强制执行这些限制。它们不仅涵盖Claude Code的直接交互，还涵盖命令产生的任何脚本、程序或子进程。

如上所述，这个沙盒强制执行：

1. **文件系统隔离**：允许对当前工作目录的读写访问，但阻止对其之外的任何文件的修改。
2. **网络隔离**：只允许通过连接到沙盒外运行的代理服务器的unix域套接字访问互联网。这个代理服务器强制执行进程可以连接的域的限制，并处理对新请求域的用户确认。

两个组件都是可配置的：你可以轻松选择允许或禁止特定文件路径或域。

沙盒确保即使成功的提示注入也完全隔离，不能影响整体用户安全。这样，受损的Claude Code不能窃取你的SSH密钥，或向攻击者的服务器打电话。

### Claude Code on the web：在云端安全运行Claude Code

今天，我们还发布了Claude Code on the web，让用户能够在隔离的沙盒中在云端运行Claude Code。Claude Code on the web在隔离的沙盒中执行每个Claude Code会话，让它以安全的方式完全访问其服务器。我们设计这个沙盒以确保敏感凭证（如git凭证或签名密钥）永远不会与Claude Code一起在沙盒内。这样，即使沙盒中运行的代码受到损害，用户也能免受进一步伤害。

Claude Code on the web使用一个自定义代理服务，透明地处理所有git交互。在沙盒内，git客户端使用自定义构建的有范围凭证向此服务进行身份验证。代理验证此凭证和git交互的内容（例如确保它只推送到配置的分支），然后在将请求发送到GitHub之前附加正确的身份验证令牌。

## 开始使用

我们的新沙盒化bash工具和Claude Code on the web为使用Claude进行工程工作的开发者提供了安全性和生产力的实质性改进。

要开始使用这些工具：

1. 在Claude中运行`/sandbox`并查看我们关于如何配置此沙盒的文档。
2. 访问claude.com/code尝试Claude Code on the web。

或者，如果你正在构建自己的智能体，请查看我们开源的沙盒代码，并考虑将其集成到你的工作中。
